**diffReviewer.ts** — Two exported functions:

1. `extractFileDiffs(projectPath: string, executionResults: ExecutionResult[], specs: SpecMetadata[]): FileDiff[]`
   - Collect all unique file paths from `executionResults[].filesChanged`
   - For each file, determine if it's a new file (appears in any spec's `affectedFiles.create`) or modified
   - For modified files: `execSync('git diff HEAD -- "filepath"', { cwd: projectPath, encoding: 'utf-8', timeout: 10000 })`. If diff is empty, try `git diff --cached -- "filepath"` as fallback
   - For new files: read full content with `fs.readFileSync(path.join(projectPath, filepath), 'utf-8')` and set `isNew: true`
   - Truncate individual diffs to 500 lines max. Log when truncation occurs
   - Normalize file paths (replace backslashes with forward slashes) per Phase 3 Windows compat pattern
   - Wrap each file in try/catch — on error, return `{ filePath, diff: '', isNew: false, error: String(error) }`

2. `reviewFileDiffs(fileDiffs: FileDiff[], specs: SpecMetadata[], reviewModel: string | null): Promise<ReviewStageResult>`
   - Build a review prompt containing: the quality rubric (3 dimensions: logic correctness, naming/conventions, type safety), each file's diff, and the spec acceptance criteria for files that match a spec's affectedFiles
   - The prompt MUST request JSON output in the shape `{ files: Array<{ filePath, passed, rationale, rubricScores: { logicCorrectness, namingConventions, typeSafety } }> }`
   - Call the LLM via `fetch('http://localhost:3000/api/ai/chat', { method: 'POST', body: JSON.stringify({ messages: [{ role: 'user', content: prompt }], model: reviewModel || 'sonnet' }) })` — this uses the existing AI chat proxy route
   - Parse the LLM response JSON. Wrap `JSON.parse` in try/catch — on parse failure, create a single "review_error" FileReviewResult with `passed: false`, rationale containing the raw LLM output, and all rubric scores as 'fail'
   - Skip files that had extraction errors (empty diff + error set) — mark them as `passed: false` with rationale "Diff extraction failed"
   - Return `ReviewStageResult` with `overallPassed = fileResults.every(r => r.passed)`, `reviewModel` string, `reviewedAt` ISO timestamp

**reportGenerator.ts** — One exported function:

`generateExecutionReport(input: ReviewStageInput, reviewResult: ReviewStageResult): ExecutionReport`
   - Derive `buildStatus` from `input.buildResult`: if skipped → 'skipped', if passed → 'passed', else → 'failed'
   - Derive `reviewOutcome`: if reviewResult exists and no error → passed/failed per overallPassed, else → 'error'
   - Derive `overallResult`: if buildStatus=passed AND reviewOutcome=passed → 'success'; if either failed → 'failure'; mixed → 'partial'
   - Collect `specTitles` from `input.specs.map(s => s.title)`
   - Collect `filesChanged` from `input.executionResults.flatMap(r => r.filesChanged || [])`  deduplicated
   - Return the full `ExecutionReport` object with `generatedAt: new Date().toISOString()`, `autoCommitted: false` (caller updates this after commit)
