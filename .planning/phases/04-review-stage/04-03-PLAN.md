**Refactor reviewStage.ts:**

Change the input type from the local `ReviewInput` to `ReviewStageInput` from `../review/reviewTypes`. The return type expands to include `reviewResults` and `report`:

```typescript
export async function executeReviewStage(input: ReviewStageInput): Promise<{
  decision: ReviewDecision;
  updatedMetrics: PipelineMetrics;
  errors: ErrorClassification[];
  reviewResults: ReviewStageResult | null;
  report: ExecutionReport | null;
}>
```

The function body becomes an orchestrator for sub-operations (preserve existing metric aggregation and decision logic):

1. **Extract diffs:** `const fileDiffs = extractFileDiffs(input.projectPath, input.executionResults, input.specs)`
2. **LLM code review:** `const reviewResults = await reviewFileDiffs(fileDiffs, input.specs, input.reviewModel)` — wrap in try/catch; on failure set `reviewResults = null` and log error. Review failure does NOT block the pipeline per locked decision.
3. **Brain signal writes per spec:** For each spec in `input.specs`, call `recordSignal()` from `@/lib/brain/brainService` with:
   - `signalType: 'implementation'` (use string literal, not enum import to avoid circular dep issues)
   - `data: { requirementId: spec.id, requirementName: spec.title, filesCreated: spec.affectedFiles.create, filesModified: spec.affectedFiles.modify, filesDeleted: spec.affectedFiles.delete, success: reviewResults?.overallPassed ?? false, executionTimeMs: 0, reviewRationale: <per-file results for this spec's files> }`
   - Wrap each signal write in try/catch — log error, don't throw. Brain is enrichment, not critical path.
   - This REPLACES the existing `recordBrainSignals()` function that uses HTTP fetch. Delete that function and the `getBaseUrl()` helper.
4. **Metric aggregation:** Preserve existing logic (completedCount, failedCount, totalDuration, updatedMetrics)
5. **Make decision:** Preserve existing `makeDecision()` logic
6. **Generate report:** `const report = generateExecutionReport(input, reviewResults || fallbackReviewResult)` — create a minimal fallback ReviewStageResult if reviewResults is null (overallPassed: false, empty fileResults)
7. **Auto-commit:** If `input.autoCommit && reviewResults && canCommit(input.buildResult, reviewResults)`, call `commitChanges(input.projectPath, input.goalTitle, input.specs.length, allFilesChanged)`. If commit succeeds, set `report.autoCommitted = true` and `report.commitSha = result.sha`.

Preserve existing `classifyError()`, `detectErrorType()`, and `makeDecision()` helper functions unchanged.
