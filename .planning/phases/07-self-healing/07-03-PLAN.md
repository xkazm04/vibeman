   c. **Add bounded retry after execute stage failures** -- In both healing trigger locations (post-scout-failure around line 388, and post-review around line 826), after classifying errors:
      ```
      const MAX_HEAL_RETRIES = 3;
      for (const result of failedResults) {
        const classification = classifyError(result.error, 'execute', runId, result.taskId);
        const retryCount = conductorRepository.getRetryCount(runId, classification.errorType, result.taskId);
        if (retryCount < MAX_HEAL_RETRIES && config.healingEnabled) {
          const patches = await analyzeErrors([classification], runId);
          activePatches.push(...patches);
          for (const p of patches) { savePatchToDb(p); }
          conductorRepository.incrementRetryCount(runId, classification.errorType, result.taskId);
          log('execute', 'info', `Retry ${retryCount + 1}/${MAX_HEAL_RETRIES} for ${classification.errorType} on task ${result.taskId}`);
          // Note: actual re-execution happens on next cycle iteration -- patches are now active
        }
      }
      ```
      The key insight: retry leverages the existing cycle loop. By adding patches and incrementing retry count, the next cycle will use healing context. This is bounded because `getRetryCount` will eventually return >= MAX_HEAL_RETRIES, stopping further patch generation.

   d. **Save classifications on run record** -- After collecting all errors in the review stage section, call:
      `conductorRepository.saveClassificationsOnRun(runId, allErrors);`
      This makes classifications visible on the run record per HEAL-01.

   e. **Update patch stats after successful run** -- At the end of a successful cycle (after commit/report), update each active patch:
      ```
      for (const patch of activePatches) {
        if (!patch.reverted) {
          updatePatchStats(patch.id, true); // success = no errors this cycle
        }
      }
      ```
      And in the failure path, update with `false`:
      ```
      for (const patch of activePatches) {
        if (!patch.reverted) {
          updatePatchStats(patch.id, false);
        }
      }
      ```
