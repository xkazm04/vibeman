  <action>
Create a polling service that:
1. Polls Supabase vibeman_commands every 60 seconds (configurable)
2. Queries for pending direction commands:
   ```typescript
   const { data: commands } = await supabase
     .from('vibeman_commands')
     .select('*')
     .in('command_type', ['accept_direction', 'reject_direction', 'skip_direction'])
     .eq('status', 'pending')
     .order('created_at');
   ```
3. Processes each command:
   - `accept_direction`: Call `directionRepository.acceptDirection(directionId, '', '')`
     - Note: requirement_id and requirement_path are empty since Butler only triages, doesn't create requirements
     - Actually use `directionRepository.updateDirection(id, { status: 'accepted' })` instead
   - `reject_direction`: Call `directionRepository.rejectDirection(directionId)`
   - `skip_direction`: No SQLite update needed (direction stays pending for re-triage later)
4. Mark command as completed in Supabase:
   ```typescript
   await supabase
     .from('vibeman_commands')
     .update({ status: 'completed', completed_at: new Date().toISOString() })
     .eq('id', command.id);
   ```
5. Track processed count for badge notification

Export:
- `class DecisionPoller` with start(), stop(), pollNow() methods
- `startDecisionPolling()` convenience function
- `stopDecisionPolling()` convenience function
