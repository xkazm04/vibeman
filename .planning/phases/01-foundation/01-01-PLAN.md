    - Assigning StageInput<'scout'> to a variable typed StageInput<'triage'> fails at compile time
    - GoalInput has: title (string), description (string), targetPaths (string[] | null), excludedPaths (string[] | null), maxSessions (number, default 2), priority ('low'|'normal'|'high'), checkpointConfig ({ triage: boolean; preExecute: boolean; postReview: boolean }), useBrain (boolean, default true)
    - PipelineRun gains goalId (string) field
    - PipelineStatus gains 'queued' value for multi-run support
  </behavior>
  <action>
Modify `src/app/features/Manager/lib/conductor/types.ts`:

1. Add per-stage input/output interfaces. For now these can be minimal placeholder shapes (the real shapes get fleshed out in later phases). Each must be structurally distinct so the discriminated union is meaningful:
   - ScoutInput: { goalId: string; projectId: string; goalInput: GoalInput }
   - ScoutResult: { gapReport: string; suggestions: string[] }
   - TriageInput: { runId: string; items: unknown[] }
   - TriageResult: { approved: string[]; rejected: string[] }
   - BatchInput: { runId: string; approved: string[] }
   - BatchDescriptor: already exists, keep it
   - ExecuteInput: { runId: string; batches: unknown[] }
   - ExecutionResult: already exists or define { taskId: string; status: string; filesModified: string[] }
   - ReviewInput: { runId: string; results: unknown[] }
   - ReviewDecision: { passed: boolean; issues: string[] }

2. Create the StageIO mapped type:
```typescript
type StageIO = {
  scout: { input: ScoutInput; output: ScoutResult };
  triage: { input: TriageInput; output: TriageResult };
  batch: { input: BatchInput; output: BatchDescriptor };
  execute: { input: ExecuteInput; output: ExecutionResult[] };
  review: { input: ReviewInput; output: ReviewDecision };
};
export type StageInput<S extends PipelineStage> = StageIO[S]['input'];
export type StageOutput<S extends PipelineStage> = StageIO[S]['output'];
export type StageFn<S extends PipelineStage> = (ctx: PipelineContext, input: StageInput<S>) => Promise<StageOutput<S>>;
```

3. Add GoalInput interface with all constraint fields per the decisions.

4. Add `goalId: string` to PipelineRun interface. Add `'queued'` to PipelineStatus union.

5. Add PipelineContext type: `{ runId: string; projectId: string; goalId: string; config: BalancingConfig; abortSignal?: { shouldAbort: boolean } }`.

6. Export all new types. Keep all existing types and exports intact -- this is additive only.
