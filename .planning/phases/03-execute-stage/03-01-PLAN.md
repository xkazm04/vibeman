  <files>src/app/features/Manager/lib/conductor/execution/fileVerifier.ts, src/app/features/Manager/lib/conductor/execution/buildValidator.ts, tests/conductor/file-verifier.test.ts, tests/conductor/build-validator.test.ts</files>
  <behavior>
    - `snapshotFiles(projectPath, paths)` returns FileSnapshot[] with exists boolean and mtimeMs for each path
    - `verifyExecution` fails if any `create` file does not exist after execution
    - `verifyExecution` fails if any `delete` file still exists after execution
    - `verifyExecution` fails if NO file in `modify` list changed mtime (at least one must have changed)
    - `verifyExecution` passes if modify list is empty and create/delete checks pass
    - `verifyExecution` passes when at least one modify file has different mtime
    - `runBuildValidation` returns `{ passed: true }` when tsc succeeds
    - `runBuildValidation` returns `{ passed: false, errorOutput }` when tsc fails
    - `runBuildValidation` returns `{ passed: true, skipped: true }` when no tsconfig.json found
    - `runBuildValidation` records durationMs in all cases
  </behavior>
  <action>
    1. Create `execution/fileVerifier.ts`:
       - `import * as fs from 'node:fs'` and `import * as path from 'node:path'`
       - Export `FileSnapshot` interface: `{ path: string, exists: boolean, mtimeMs: number }`
       - Export `VerificationResult` interface: `{ passed: boolean, reason?: string }`
       - Export `snapshotFiles(projectPath: string, paths: string[]): FileSnapshot[]` — use `fs.statSync` with try/catch, return `{ exists: false, mtimeMs: 0 }` on error
       - Export `verifyExecution(projectPath: string, affectedFiles: AffectedFiles, beforeSnapshots: FileSnapshot[]): VerificationResult`:
         - Check `create` files exist (fs.existsSync)
         - Check `delete` files do NOT exist (fs.existsSync)
         - Check `modify` files: at least one must have mtimeMs different from beforeSnapshot. Use mtime comparison (per user discretion — mtime is sufficient)
         - Return `{ passed: true }` if all checks pass
         - Return `{ passed: false, reason: '...' }` with specific failure message

    2. Create `execution/buildValidator.ts`:
       - `import { execSync } from 'child_process'` and `import * as fs from 'node:fs'` and `import * as path from 'node:path'`
       - Export `BuildResult` interface: `{ passed: boolean, errorOutput?: string, durationMs: number, skipped?: boolean, reason?: string }`
       - Export `runBuildValidation(projectPath: string): BuildResult`:
         - First check `fs.existsSync(path.join(projectPath, 'tsconfig.json'))` — if missing, return `{ passed: true, skipped: true, reason: 'No tsconfig.json found', durationMs: 0 }`
         - Run `execSync('npx tsc --noEmit', { cwd: projectPath, encoding: 'utf-8', timeout: 120000, stdio: ['pipe', 'pipe', 'pipe'] })`
         - On success: `{ passed: true, durationMs }`
         - On error: `{ passed: false, errorOutput: error.stderr || error.stdout || String(error), durationMs }`

    3. Create test files:
       - `tests/conductor/file-verifier.test.ts`: Use `node:fs` to create temp files, test snapshot before/after modification, test create/delete/modify verification scenarios. Use `vi.spyOn` on fs for edge cases.
       - `tests/conductor/build-validator.test.ts`: Mock `child_process.execSync` and `fs.existsSync` via vi.mock. Test success, failure with error output, missing tsconfig skip, and timeout scenarios.
